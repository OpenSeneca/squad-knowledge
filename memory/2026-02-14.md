## Technical Decisions

### WebSocket vs Polling
**Decision:** WebSocket for real-time updates

**Rationale:**
- Push notifications eliminate redundant requests
- Sub-second latency vs 30-second polling
- Better UX (no loading states)
- Lower server load
- Industry standard for dashboards

### Python f-String Templates
**Decision:** Built-in f-strings for agt templates

**Rationale:**
- No external templating engine
- Easier to read and maintain
- Simpler build process
- Zero dependencies

### Abstract Base Class Pattern
**Decision:** ABC pattern for agent scaffolding

**Rationale:**
- Enforces contract implementation
- Clear API documentation
- Type safety
- Industry best practice

## Learnings

### WebSocket Implementation
1. **Broadcast pattern:** Services accept callbacks to notify WebSocket server
2. **React hooks:** Custom hooks make WebSocket logic reusable
3. **Type safety:** TypeScript interfaces for all messages
4. **Reconnection:** Automatic retry with exponential backoff is robust
5. **Separation of concerns:** Services don't know about WebSocket

### Agent Scaffolding
1. **Templates matter:** Well-designed templates speed development
2. **Documentation is key:** README, setup instructions critical
3. **Testing included:** Test templates set quality standard
4. **Cross-platform:** pathlib vs os.path for Windows compatibility
5. **Configuration management:** Environment variables are production standard

### Framework Comparison
1. **CrewAI:** YAML config, enterprise focus
2. **AgentGPT:** Full-stack, browser-based
3. **SuperAGI:** Dev-first, modular
4. **My approach:** CLI-first, lightweight, Python
