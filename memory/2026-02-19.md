## Self-Directed Project: GitHub Release Monitor (TODO #5)

**Status:** âœ… COMPLETED

Built `gh-release-monitor` CLI tool to track GitHub releases without notification noise.

**Why This Tool:**

While researching trending AI tools, I found that:
- GitHub notifications are too noisy for tracking specific repos
- No simple CLI exists to monitor releases from multiple repos
- Justin writes about AI and needs to track new SDK/tool releases
- This is something he'd run daily/weekly

**Features:**
- Monitor multiple repositories via CLI args or config file
- Config file support (one repo per line, comments allowed)
- Date filtering (`--since YYYY-MM-DD`)
- Keyword filtering (`--keywords "api gpt"`)
- Multiple output formats:
  - Text (human-readable)
  - JSON (machine-readable for automation)
  - Markdown (blog-friendly)
- Include release notes (`--show-body`)
- 5-minute caching (reduces API calls)
- Uses `gh` CLI for authentication
- Prerelease/Draft badges

**Usage Examples:**

```bash
# Monitor specific repos
gh-release-monitor --repos openai/openai-python langchain-ai/langchain

# Use config file
gh-release-monitor --config ai-tools.txt --since 2026-02-15

# Filter by keywords
gh-release-monitor --config ai-tools.txt --keywords "api feature"

# Markdown output for blog posts
gh-release-monitor --config ai-tools.txt --markdown --show-body > blog.md
```
